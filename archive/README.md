# deepface-rs — DeepFace Core Logic in Rust (Axum + ONNX)

## What This Is

A complete port of DeepFace's core face verification pipeline from Python/TensorFlow
to Rust using Axum (web server) + ort (ONNX Runtime).

**Python DeepFace call being replaced:**
```python
result = DeepFace.verify(
    img1_path="face1.jpg",
    img2_path="face2.jpg",
    model_name="Facenet512",
    detector_backend="mtcnn",
    distance_metric="cosine",
    align=True
)
```

## Why Rust Instead of Python

| | Python (DeepFace) | Rust (this) |
|---|---|---|
| Cold start time | 4–8 seconds (TF init) | ~50ms (ONNX session reused) |
| RAM per request | 1–4 GB (TF graph) | ~120 MB (model loaded once) |
| Requests/sec | ~5–10 (GIL limited) | ~200–500 (fully async) |
| Inference precision | f32 | f32 (identical) |
| Accuracy | 99.6% (LFW benchmark) | Identical (same model weights) |

## Project Structure

```
deepface-rs/
├── Cargo.toml              — dependencies
├── export_to_onnx.py       — run once to get .onnx files from DeepFace
├── models/                 — ONNX model files (generated by export script)
│   ├── facenet512.onnx
│   ├── mtcnn_pnet.onnx
│   ├── mtcnn_rnet.onnx
│   └── mtcnn_onet.onnx
└── src/
    ├── main.rs             — Axum server, AppState, routing, pipeline orchestration
    ├── mtcnn.rs            — MTCNN 3-stage detection (P-Net → R-Net → O-Net + NMS)
    ├── alignment.rs        — Eye-based affine rotation (align=True logic)
    ├── preprocessing.rs    — Crop to bbox, resize to 160x160, FaceNet normalization
    ├── inference.rs        — FaceNet512 ONNX inference → 512-dim embedding
    ├── math.rs             — L2 normalization + cosine distance
    └── errors.rs           — Unified error type → HTTP responses
```

## Step 1: Export ONNX Models (One Time, on any machine with Python)

```bash
pip install deepface tensorflow tf2onnx onnx mtcnn
python export_to_onnx.py
# This produces 4 .onnx files in ./models/
```

You only need Python for this step. After this, Python is never needed again.

## Step 2: Build and Run (Rust only)

```bash
# Install ORT native library first:
# https://onnxruntime.ai/docs/install/

cargo build --release
./target/release/deepface-rs
# Server starts on 0.0.0.0:8080
```

## API Usage

### JSON Endpoint
```bash
curl -X POST http://localhost:8080/api/v1/verify \
  -H "Content-Type: application/json" \
  -d '{
    "image1_b64": "<base64_encoded_jpeg>",
    "image2_b64": "<base64_encoded_jpeg>"
  }'
```

### Multipart Endpoint
```bash
curl -X POST http://localhost:8080/api/v1/verify_multipart \
  -F "image1=@face1.jpg" \
  -F "image2=@face2.jpg"
```

### Response
```json
{
  "verified": true,
  "distance": 0.18,
  "threshold": 0.30,
  "model": "Facenet512",
  "detector": "MTCNN",
  "metric": "cosine"
}
```

## The Full Pipeline (What Happens Per Request)

```
Raw image bytes
      ↓
image::load_from_memory()       ← replaces cv2.imread
      ↓
mtcnn::detect_face()            ← P-Net (pyramid) → NMS → R-Net → NMS → O-Net → landmarks
      ↓
alignment::align_face()         ← rotate image so eyes are horizontal (align=True)
      ↓
preprocessing::crop_and_resize()← crop bbox, resize to 160×160 (bicubic)
      ↓
preprocessing::normalize_facenet()← per-image (pixel-mean)/std  ← CRITICAL
      ↓
inference::get_embedding()      ← FaceNet512 ONNX → [1, 512] f32
      ↓
math::l2_normalize()            ← unit vector ||v|| = 1
      ↓
math::cosine_distance()         ← 1 - dot(embed1, embed2)
      ↓
distance <= 0.30 → verified: true
```

## Precision Notes

- **NO INT8 quantization** — we use f32 throughout, same as Python DeepFace
- **Normalization must match training** — FaceNet512 uses per-image z-score ("Facenet" mode)
  Using "base" (/255) or "Facenet2018" ([-1,1]) gives wrong results with the same model
- **MTCNN landmark order** — [left_eye, right_eye, nose, mouth_left, mouth_right]
- **ONNX layout** — FaceNet512 input is NHWC [1,160,160,3]; MTCNN inputs are NCHW [1,3,H,W]
